LSM-Tree Component: Compaction Strategy

กลยุทธ์การบริหารจัดการไฟล์ SSTables บน Disk เพื่อประสิทธิภาพระยะยาว

A

Leveled Compaction

ลดการซ้ำซ้อนของข้อมูล (Space Amplification) เหมาะกับ Read-heavy

Style: RocksDB
B

Size-Tiered Compaction

เน้นความเร็วในการเขียน (Write-heavy) เป็นหลัก

Style: Cassandra / ScyllaDB
C

Time-Window Compaction

เน้นการ Query ตามช่วงเวลาและการลบข้อมูลเก่า (TTL) ได้ง่ายที่สุด

Best for: Time-Series / IoT / Logs
